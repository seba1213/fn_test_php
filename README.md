# Ответы

1. MySql предпочтителен, однако взаимодействие осуществляем через Eloquent
2. Создавать ссылки с одинаковыми исходными данными не запрещено, статистика ведется по обращению к короткой ссылке
3. Для проверки существования исходных ссылок можно использовать head запрос, валидация - соответствует ли текст ссылки rfc 3986
4. Парольная аутентификация
5. Выбор реквизитов (логин/email) по желанию, валидация на корректность и уникальность. В рамках тз к паролю требований нет (только наличие)
6. 2fa не требуется
7. Использование идентификаторов бд не желательно, генерация последовательности символов [a-z0-9]
8. Код редиректа в рамках тз не принципиален
9. Заглушка вида "Запрашиваемая ссылка отсутствует"
10. Статистика переходов хранится в основной базе данных
11. SoftDelete не использовать, статистика удаляется вместе со ссылкой


# Замечания по реализации

## Формирование кода для короткой ссылки 

1. Сначала создаётся запись в таблице `short_links` с временным случайным кодом.
2. После получения уникального идентификатора (ID) записи, для него формируется короткий код — это base36-представление числа (ID + 10000). Таким образом, итоговая короткая ссылка — это строка, состоящая из символов латинского алфавита и цифр, например: `7n7t`, `ac42`, и т.п.

Использую идентификатор из базы данных как основу так как не используется SoftDelete и есть вероятность генерации ссылки которая ранее была создана и удалена.

## Как зайти в панель из devcontainer

3. Перейдите по адресу [http://localhost:8080/admin](http://localhost:8080/admin) в браузере — панель Filament доступна по этому пути.
